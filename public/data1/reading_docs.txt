# IELTS Reading JSON Schema Documentation

## Overview
This document defines the JSON structure used to represent an IELTS Reading test. The schema is highly modular, separating the reading passage into distinct blocks and handling various IELTS question types within a unified `questions` array. 

---

## 1. Root Object: The Reading Passage
The top level of the JSON object defines the metadata and the text of the reading passage itself, along with the array of questions.

* **`id`** (string): A unique identifier for the passage (e.g., "passage_green_roofs").
* **`module`** (string): The test module (e.g., "reading").
* **`passage_number`** (integer): The standard IELTS passage number (1, 2, or 3).
* **`title`** (string): The main title of the passage.
* **`subtitle`** (string): The subtitle of the passage (can be empty `""`).
* **`instruction`** (string): The overarching instruction for the passage (e.g., "You should spend about 20 minutes...").
* **`word_count`** (integer): Total word count of the passage.
* **`difficulty`** (string): Difficulty level (e.g., "easy", "medium", "hard").
* **`blocks`** (array of objects): The reading passage text, split into paragraphs.
    * `type` (string): The type of content (e.g., "text", "image", "table").
    * `label` (string): The paragraph letter (e.g., "A", "B").
    * `content` (string): The actual text of the paragraph.
* **`questions`** (array of objects): An array containing different question blocks. Each block represents a specific IELTS question type.

---

## 2. Question Types

Every question block in the `questions` array shares a few standard keys:
* **`type`** (string): Defines the question format.
* **`start_question`** (integer): The first question number in this block.
* **`end_question`** (integer): The last question number in this block.
* **`instructions`** (string): The specific instructions for this question block.

Below are the specific structures for the supported question types.

### Type A: Matching Headings (`"type": "matching_headings"`)
Used when users must match Roman numeral headings to paragraphs.

* **`options`** (array of objects): The list of available headings.
    * `id` (string): The Roman numeral (e.g., "i", "ii").
    * `text` (string): The text of the heading.
* **`questions`** (array of objects): The items to evaluate.
    * `question_number` (integer): The question number.
    * `target_paragraph` (string): The paragraph label the user is evaluating (e.g., "B").
    * `correct_answer` (string): The correct Roman numeral (e.g., "iv").

### Type B: True / False / Not Given OR Yes / No / Not Given (`"type": "true_false_not_given"`)
Used for evaluating statements against the text. Works for both TFNG and YNNG depending on how the `options` are populated.

* **`options`** (array of objects): The three fixed choices.
    * `label` (string): "TRUE", "FALSE", "NOT GIVEN" (or "YES", "NO").
    * `description` (string): The explanation of the label (e.g., "if the statement agrees...").
* **`questions`** (array of objects): The statements to evaluate.
    * `question_number` (integer): The question number.
    * `statement` (string): The sentence the user must evaluate.
    * `correct_answer` (string): Must exactly match one of the option `label`s (e.g., "TRUE").
    * `passage_reference` (string): The paragraph label where the answer is found.

### Type C: Matching Paragraph Information (`"type": "matching_paragraph_information"`)
Used when users must find which paragraph contains specific information. 

* **`options`** (array of strings): A simple list of available paragraph letters (e.g., `["A", "B", "C", "D", "E"]`).
* **`questions`** (array of objects): The information the user needs to locate.
    * `question_number` (integer): The question number.
    * `statement` (string): The piece of information to find.
    * `correct_answer` (string): The exact paragraph letter (e.g., "C").

### Type D: Summary Completion (`"type": "summary_completion"`)
Used for fill-in-the-blank summaries. This schema seamlessly handles both "Direct Entry" (typing words) and "Word Bank" (choosing letters).

* **`summary_title`** (string): The header text for the summary.
* **`summary_text`** (string): The paragraph text. **Crucial:** Blanks are represented by brackets containing the question number (e.g., `[5]`, `[6]`).
* **`options`** (null OR array of objects): 
    * If **`null`**: This is a Direct Entry task.
    * If **populated** (e.g., `[{"id": "A", "text": "jobs"}]`): This is a Word Bank task.
* **`questions`** (array of objects): The answers for the blanks.
    * `question_number` (integer): Matches the bracket in `summary_text`.
    * `correct_answer` (array of strings): A list of all acceptable exact-match answers to accommodate IELTS grading rules (e.g., `["10%", "10 percent"]`). For single text answers or word bank letters, use an array with one item (e.g., `["floods"]` or `["C"]`).
    * `passage_reference` (string): The paragraph label where the answer is sourced.

### Type E: Sentence Completion (`"type": "sentence_completion"`)
Used when users must complete standalone sentences using words directly from the text.

* **`questions`** (array of objects): The sentences to complete.
    * `question_number` (integer): The question number.
    * `sentence` (string): The sentence text. **Crucial:** The blank is represented by brackets containing the question number (e.g., `[8]`).
    * `correct_answer` (array of strings): A list of all acceptable exact-match answers (e.g., `["factory", "plant"]`). If there is only one acceptable answer, use an array with one item (e.g., `["weight"]`).
    * `passage_reference` (string): The paragraph label where the answer is found.

### Type F: Multiple Choice (`"type": "multiple_choice"`)
Used for standard multiple-choice questions. Note that the `options` array is located *inside* each question object, rather than at the root level, because options vary per question.

* **`questions`** (array of objects): The multiple-choice questions.
    * `question_number` (integer): The question number.
    * `question_text` (string): The actual question or incomplete sentence.
    * `options` (array of objects): The specific choices for this question.
        * `id` (string): The option letter (e.g., "A", "B", "C", "D").
        * `text` (string): The text of the option.
    * `correct_answer` (array of strings): The ID(s) of the correct choice(s). Formatted as an array to accommodate standard 1-answer questions (e.g., `["C"]`) as well as IELTS "Choose TWO letters" variations (e.g., `["A", "E"]`).
    * `passage_reference` (string): The paragraph label where the answer is found.

### Type H: List Selection (`"type": "list_selection"`)
Used when users must select multiple options from a list for a single prompt (e.g., "Choose TWO letters, A-E"). Note that a single question object covers multiple question numbers (as defined by `start_question` and `end_question`).

* **`questions`** (array of objects): The selection question block.
    * `question_number` (integer): The starting question number for this specific prompt (e.g., 18).
    * `question_text` (string): The prompt asking the user what to select.
    * `options` (array of objects): The specific choices for this question.
        * `id` (string): The option letter (e.g., "A", "B", "C", "D", "E").
        * `text` (string): The text of the option.
    * `correct_answer` (array of strings): The IDs of all the correct choices, formatted as an array (e.g., `["B", "D"]`). The number of items in this array corresponds to the number of choices the user needs to make.
    * `passage_reference` (string): The paragraph label where the answers are found.

### Type G: Matching Features / Classification (`"type": "matching_features"`)
Used when users must categorize statements or match them to a shared list of options (e.g., Classifying information into Categories A, B, or C, or matching to a List of Researchers). 

* **`options`** (array of objects): The shared categories or choices available for all questions in this block.
    * `id` (string): The option letter (e.g., "A", "B", "C").
    * `text` (string): The text of the category/option.
* **`questions`** (array of objects): The statements the user must classify.
    * `question_number` (integer): The question number.
    * `statement` (string): The statement that needs to be categorized.
    * `correct_answer` (array of strings): The ID of the correct category, formatted as an array (e.g., `["A"]`).
    * `passage_reference` (string): The paragraph label where the connection is found in the text.

### Type I: Matching Sentence Endings (`"type": "matching_sentence_endings"`)
Used when users must complete a sentence by choosing the correct ending from a shared list of options. 

* **`options`** (array of objects): The shared list of sentence endings available for all questions in this block.
    * `id` (string): The option letter (e.g., "A", "B", "C").
    * `text` (string): The text of the sentence ending.
* **`questions`** (array of objects): The incomplete sentences the user must finish.
    * `question_number` (integer): The question number.
    * `sentence_start` (string): The first half of the sentence that needs an ending.
    * `correct_answer` (array of strings): The ID of the correct ending, formatted as an array (e.g., `["A"]`).
    * `passage_reference` (string): The paragraph label where the connection is found in the text.

### Type J: Table Completion (`"type": "table_completion"`)
Used when users must complete a grid/table by filling in blanks. This type supports both direct text entry and word banks.

* **`table_title`** (string): The title/heading of the table (can be an empty string `""` if the table has no title).
* **`options`** (null OR array of objects): 
    * If **`null`**: Direct Entry task (user types the word).
    * If **populated**: Word Bank task (user selects a letter ID).
* **`headers`** (array of strings): The column titles for the table (e.g., `["Type", "Benefit", "Challenge"]`).
* **`rows`** (array of arrays of strings): The data rows of the table. Each array represents one row. **Crucial:** Blanks within the cells are represented by brackets containing the question number (e.g., `[23]`).
* **`questions`** (array of objects): The answers for the blanks.
    * `question_number` (integer): Matches the bracket in the `rows` cell.
    * `correct_answer` (array of strings): A list of all acceptable exact-match answers (e.g., `["floods", "flood damage"]`). If there is a Word Bank, this is an array containing the single correct letter ID (e.g., `["A"]`).
    * `passage_reference` (string): The paragraph label where the answer is found.

### Type K: Flow Chart Completion (`"type": "flowchart_completion"`)
Used when users must complete a sequence of steps describing a process. This type supports both direct text entry and word banks.

* **`flowchart_title`** (string): The title/heading of the flow chart (can be an empty string `""` if there is no title).
* **`options`** (null OR array of objects): 
    * If **`null`**: Direct Entry task (user types the word).
    * If **populated**: Word Bank task (user selects a letter ID).

### Type L: Diagram Completion (`"type": "diagram_completion"`)
Used when users must label parts of an image/diagram using words from the text. Note: In this format, the contextual text, lines, and pointers are assumed to be part of the image file itself.

* **`diagram_title`** (string): The title of the diagram (can be `""` if there is no title).
* **`diagram_url`** (string): The file path or URL to the diagram image that the front-end needs to render.
* **`diagram_alt_text`** (string): Accessibility text describing the image.
* **`options`** (null OR array of objects): 
    * If **`null`**: Direct Entry task (user types the word).
    * If **populated**: Word Bank task (user selects a letter ID).
* **`questions`** (array of objects): The input requirements for the diagram. The front-end should render simple input fields mapped to these numbers alongside or below the image.
    * `question_number` (integer): The question number that corresponds to a label in the diagram image.
    * `correct_answer` (array of strings): A list of all acceptable exact-match answers (e.g., `["vegetation", "plants"]`). If there is a Word Bank, this is an array containing the single correct letter ID.
    * `passage_reference` (string): The paragraph label where the answer is found.

### Type M: Short Answer Questions (`"type": "short_answer"`)
Used when users must write a short answer (extracted directly from the passage) to a direct question.

* **`questions`** (array of objects): The list of direct questions.
    * `question_number` (integer): The question number.
    * `question_text` (string): The direct question being asked.
    * `correct_answer` (array of strings): A list of all acceptable exact-match answers extracted from the text (e.g., `["floods", "flood damage"]`).
    * `passage_reference` (string): The paragraph label where the answer is found.

---

## Instructions for LLM Generation
When generating new IELTS reading passages and questions using this schema:
1. Ensure `start_question` and `end_question` accurately reflect the `question_number` items within the block.
2. Ensure continuous global question numbering across the entire test (e.g., Block 1 is 1-5, Block 2 is 6-9, etc.).
3. Maintain exact string matching between `correct_answer` and the respective `options` IDs or Labels.
4. For `summary_completion` and `sentence_completion`, ensure the bracketed numbers in the text strings (e.g., `[5]`) perfectly match the `question_number` in the `questions` array.
5. Always format the `correct_answer` for completion types (Type D and Type E) as an **array of strings**, providing valid alternative formats or synonyms found strictly in the passage.
