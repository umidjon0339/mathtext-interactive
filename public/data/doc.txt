K-12 DIGITAL BOOK JSON STRUCTURE SPECIFICATION (v2.1)
=========================================================

This is a universal, stack-agnostic JSON structure for a multilingual
K-12 digital textbook. It is designed to be rendered identically in any
frontend framework (Jetpack Compose, Next.js/React, etc.). The structure
separates pure educational content from presentation, supports multiple
languages, and uses a block-based architecture with structured inline text.

---------------------------------------------------------
1. OVERALL LESSON / SECTION JSON
---------------------------------------------------------
Every lesson, section, or page is a JSON object with these top-level fields:

{
  "id": "9.1",                           // Unique identifier (string)
  "title": {                             // Multilingual string (see §2)
    "en": "Properties of Radicals",
    "uz": "Ildizlarning xossalari",
    "ru": "Свойства радикалов"
  },
  "langs": ["en", "uz", "ru"],           // Available languages (optional)
  "blocks": [ ... ]                      // Ordered list of content blocks (see §4)
}

---------------------------------------------------------
2. MULTILINGUAL STRINGS
---------------------------------------------------------
Every user-visible string is a map from language code to text:
{ "en": "text", "uz": "...", "ru": "..." }

The client selects the active language once and picks the corresponding value.
If a language is missing, fall back to English.

---------------------------------------------------------
3. INLINE RICH TEXT FORMAT
---------------------------------------------------------
All text is stored NOT as raw strings with markup, but as an array of 
inline objects. This is the ONLY way text appears in the entire structure.

An inline array is an ordered list of nodes, where each node has a "type":

- plain: Simple text
  { "type": "plain", "text": "Write and solve " }

- term: Glossary term (clickable, links to glossary)
  { "type": "term", "id": "equation", "display": "equations" }
  "id" is the key in the global glossary, "display" is the visible text.

- math: LaTeX inline math (render with KaTeX/MathJax)
  { "type": "math", "latex": "\\sqrt{a}" }

- bold, italic, underline: Formatting that wraps other inline nodes
  { "type": "bold", "text": [ { "type": "plain", "text": "strong text" } ] }
  Same structure for italic and underline.

- highlight: Visually highlighted text (like <mark>)
  { "type": "highlight", "text": [ ... ] }

Example of a full inline text:
"text": [
  { "type": "plain", "text": "Write and solve " },
  { "type": "term", "id": "equation", "display": "equations" },
  { "type": "plain", "text": " with " },
  { "type": "term", "id": "variable", "display": "variables" },
  { "type": "plain", "text": " on both sides." }
]

IMPORTANT: No {{...}} or <Word ...> markup anywhere. All formatted text 
must be converted to this array format.

---------------------------------------------------------
4. TOP-LEVEL BLOCKS (DISCRIMINATED UNION)
---------------------------------------------------------
The "blocks" array contains objects, each with a "type" field. The 
rest of the structure depends on the type. These are the block types:

4.1 learning_target
--------------------
{
  "type": "learning_target",
  "target": [ Inline array ],                // Learning objective
  "success_criteria": [                      // Array of inline arrays
    [ Inline array ],
    [ Inline array ],
    ...
  ]
}

4.2 explore_it
--------------
{
  "type": "explore_it",
  "header": {
    "title": { Multilingual string }
  },
  "margin_content": [                        // Items rendered in side margin
    { type: "mtr_badge", ... },
    { type: "image", ... }
  ],
  "body": [                                  // Main content blocks (task_blocks, etc.)
    { type: "instruction_text", ... },
    { type: "task_block", ... },
    ...
  ]
}

4.3 lesson_concept
------------------
{
  "type": "lesson_concept",
  "id": "sec_03",                            // Anchor ID (optional)
  "title": { Multilingual string },
  "margin_content": [                        // Usually vocabulary_box, etc.
    { type: "vocabulary_box", ... }
  ],
  "body": [                                  // paragraphs, bullet_lists, key_ideas_box, etc.
    { type: "paragraph", ... },
    { type: "key_ideas_box", ... }
  ]
}

4.4 example_block
-----------------
{
  "type": "example_block",
  "id": "sec_06",                            // Optional anchor
  "has_video": true,                         // Boolean flag
  "label": { Multilingual string },         // e.g. "EXAMPLE 2"
  "title": { Multilingual string },
  "margin_content": [                        // Usually study_tip_box
    { type: "study_tip_box", ... }
  ],
  "body": [
    { type: "paragraph", ... },
    { type: "worked_solutions", ... }
  ]
}

4.5 self_assessment_block
-------------------------
{
  "type": "self_assessment_block",
  "id": "sec_07_self_assessment",            // Anchor ID
  "title": { Multilingual string },
  "rating_scale": [                          // 4-level scale
    { "value": 1, "text": { Multilingual } },
    { "value": 2, "text": { Multilingual } },
    { "value": 3, "text": { Multilingual } },
    { "value": 4, "text": { Multilingual } }
  ],
  "body": [
    { "type": "instruction_text", ... },
    { "type": "question_grid", ... }
  ]
}

---------------------------------------------------------
5. GENERIC / NESTABLE BLOCK TYPES
---------------------------------------------------------
These can appear inside "body", "content", or "margin_content" arrays.

5.1 paragraph
--------------
{ "type": "paragraph", "text": [ Inline array ] }

5.2 instruction_text
---------------------
Same as paragraph but typically used for teacher instructions.
{ "type": "instruction_text", "text": [ Inline array ] }

5.3 bullet_list
----------------
{ "type": "bullet_list", "items": [ [Inline array], [Inline array], ... ] }

5.4 equation_grid
------------------
Displays a grid of LaTeX equations. 
{ 
  "type": "equation_grid",
  "columns": 1,           // number of columns (hint for layout)
  "align": "left",        // "left" or "center"
  "items": [
    { "id": "a_i", "latex": "\\text{i. } \\sqrt{a} + \\sqrt{b} ..." },
    ...
  ]
}

5.5 question_grid
------------------
Similar to equation_grid but for self‑assessment questions.
{
  "type": "question_grid",
  "columns": 2,
  "items": [
    { "id": "sa_q1", "label": "1.", "latex": "\\sqrt{24}" },
    ...
  ]
}

5.6 card_tiles
---------------
Visual cards with LaTeX content.
{
  "type": "card_tiles",
  "tiles": [
    { "id": "t1", "latex": "2" },
    { "id": "t2", "latex": "\\frac14" }
  ]
}

5.7 worked_solutions
---------------------
Step-by-step worked example.
{
  "type": "worked_solutions",
  "steps": [
    {
      "label": "a.",                        // Optional step label
      "expression": "\\sqrt[3]{-128}",      // Initial expression
      "work": [
        {
          "line": "= \\sqrt[3]{-64 \\cdot 2}",
          "explanation": [ Inline array ]    // Why this step was taken
        },
        ...
      ]
    }
  ]
}

5.8 key_ideas_box
------------------
Highlighted box with key mathematical properties. Concepts can contain
either a simple `matrix` of rows (for quick property tables) or a rich
`content` array (for mixing property matrices, paragraphs, bullet lists, etc.).

Using `matrix` (simple, backward-compatible):
{
  "type": "key_ideas_box",
  "title": { Multilingual string },
  "concepts": [
    {
      "subtitle": { Multilingual string },
      "matrix": [
        {
          "label": { Multilingual },
          "text": [ Inline array ]           // OR "latex": "..."
        },
        {
          "label": { Multilingual },
          "latex": "..."
        },
        {
          "label": { Multilingual },
          "translations": {
            "en": { "latex": "..." },
            "uz": { "latex": "..." },
            "ru": { "latex": "..." }
          }
        }
      ]
    }
  ]
}

Using `content` (flexible, can embed any block types):
{
  "type": "key_ideas_box",
  "title": { Multilingual string },
  "concepts": [
    {
      "subtitle": { Multilingual string },
      "content": [
        { "type": "property_matrix", "rows": [ ... ] },
        { "type": "paragraph", "text": [ Inline array ] },
        { "type": "bullet_list", "items": [ ... ] }
      ]
    }
  ]
}

* A concept may have either `matrix` or `content`, but not both.
* When `content` is present, it takes precedence.
* `property_matrix` is a new block type (see §5.15) used within `content`.

5.9 vocabulary_box
-------------------
A margin box listing glossary terms.
{
  "type": "vocabulary_box",
  "icon": "az_vocab",                       // Icon identifier
  "title": { Multilingual },
  "terms": [
    {
      "term": { Multilingual },
      "reference": "p. 470"                 // Cross-reference
    }
  ]
}

5.10 study_tip_box
-------------------
Margin tip for students.
{
  "type": "study_tip_box",
  "icon": "lightbulb",
  "title": { Multilingual },
  "text": [ Inline array ]
}

5.11 mtr_badge
---------------
A "Mathematical Teaching Routine" badge in the margin.
{
  "type": "mtr_badge",
  "id": 1,
  "badge": "ANALYZE A PROBLEM",             // Non-translatable label
  "prompt": [ Inline array ]               // Guiding question
}

5.12 image
-----------
{
  "type": "image",
  "src": "/assets/graphics/student_thinking_cutout.png",
  "style": "transparent_cutout",           // Optional rendering hint
  "alt": { Multilingual string }
}

5.13 video
-----------
{
  "type": "video",
  "src": "url-to-video.mp4",
  "poster": "thumbnail.jpg"                // Optional
}

5.14 task_block
----------------
Used inside explore_it or anywhere to present a step-by-step task.
{
  "type": "task_block",
  "label": "a.",                           // Optional letter/number
  "level": 1,                              // Nesting depth: 1 = top-level, 2 = sub-task, etc.
  "instruction": [ Inline array ],
  "content": [ ... ]                       // Optional sub-blocks (equation_grid, card_tiles, etc.)
}

* For proper indentation, a task with "level": 2 (or higher) should be placed
  inside the "content" array of its parent task. The renderer uses CSS nesting to
  automatically align labels and text at the correct depth.

5.15 property_matrix
---------------------
A compact table of mathematical properties (Words, Numbers, Algebra).
Used inside key_ideas_box "content" arrays or anywhere a property list is needed.
{
  "type": "property_matrix",
  "rows": [
    {
      "label": { Multilingual },            // e.g. "Words"
      "text": [ Inline array ]              // inline text (or use "latex" / "translations")
    },
    {
      "label": { Multilingual },            // e.g. "Numbers"
      "latex": "\\sqrt{9 \\cdot 5} = 3\\sqrt{5}"
    },
    {
      "label": { Multilingual },            // e.g. "Algebra"
      "translations": {
        "en": { "latex": "..." },
        "uz": { "latex": "..." },
        "ru": { "latex": "..." }
      }
    }
  ]
}

---------------------------------------------------------
6. LAYOUT HINTS
---------------------------------------------------------
The JSON separates main content ("body") from marginal content ("margin_content").
On wide screens (desktop), margin_content is rendered in a side column; on narrow 
screens (mobile), it may be collapsed, shown as pop‑ups, or injected into the flow.

Grid blocks (equation_grid, question_grid) include "columns" as a suggestion; 
the renderer may reduce columns on small screens.
"align" in equation_grid suggests left or center alignment.

---------------------------------------------------------
7. GLOSSARY AND TERM REFERENCES
---------------------------------------------------------
The "term" inline node references a global glossary dictionary. The client 
should maintain a mapping from "id" to definitions. Example:

Glossary:
{
  "equation": {
    "id": "equation",
    "word": { "en": "equation", "uz": "tenglama", "ru": "уравнение" },
    "definition": {
      "en": "A mathematical statement that two expressions are equal.",
      "uz": "...",
      "ru": "..."
    },
    "images": [],
    "examples": {
      "en": ["2x + 3 = 11"],
      "uz": ["2x + 3 = 11"],
      "ru": ["2x + 3 = 11"]
    },
    "relatedConcepts": ["variable", "equality", "solution"]
  },
  "variable": { ... }
}

When rendering a term, the client can show a tooltip or link to the glossary.

---------------------------------------------------------
8. MATH / LATEX
---------------------------------------------------------
All math is stored as LaTeX strings in "latex" fields. Inline math uses 
the "math" inline node. Render using KaTeX, MathJax, or native library.

---------------------------------------------------------

  "blocks": [
    {
      "type": "learning_target",
      "target": { "en": [ ... ], "uz": [ ... ], "ru": [ ... ] },

    },
    {
      "type": "explore_it",
      "header": { "title": { "en": "Performing Operations with Square Roots" } },
      "margin_content": [
        {
          "type": "mtr_badge",
          "id": 1,
          "badge": "ANALYZE A PROBLEM",
          "prompt": { "en": [ ... ] }
        },
        { "type": "image", "src": "...", "alt": {...} }
      ],
      "body": [
        { "type": "instruction_text", "text": { "en": [ ... ] } },
        { "type": "task_block", "label": "a.", "level": 1, ... },
        { "type": "task_block", "label": "b.", "level": 1, ... }
      ]
    },
    {
      "type": "lesson_concept",
      "title": { "en": "Using Properties of Radicals" },

        { "type": "vocabulary_box", ... }

          "type": "key_ideas_box",

          "concepts": [
            {
              "subtitle": { "en": "Product Property of Square Roots" },

                { "type": "property_matrix", "rows": [ ... ] },

      "type": "example_block",
      "label": { "en": "EXAMPLE 2" },
      "title": { "en": "Using Properties of Cube Roots" },

        { "type": "paragraph", ... },

      "type": "self_assessment_block",
      "title": { "en": "SELF-ASSESSMENT" },
      "rating_scale": [ ... ],

---------------------------------------------------------
10. MIGRATION FROM OLD FORMAT
---------------------------------------------------------
If you have existing JSON with `{{...}}` or `<Word id="...">...</Word>` markup:

- Replace every `{{glossary_id|display_text}}` with a "term" inline node:
  { "type": "term", "id": "glossary_id", "display": "display_text" }
- Replace `<Word id="...">display</Word>` similarly.
- Split plain text around those markers into separate "plain" nodes.
- Ensure all inline content becomes an array. Example:
  Old: "Write and solve {{equation|equations}} with {{variable|variables}}"
  New: [
    { "type": "plain", "text": "Write and solve " },
    { "type": "term", "id": "equation", "display": "equations" },
    { "type": "plain", "text": " with " },
    { "type": "term", "id": "variable", "display": "variables" }
  ]

- For HTML-like tags (e.g., <em>, <strong>), use "italic" or "bold" inline types.

---------------------------------------------------------
11. RENDERING GUIDELINES FOR DEVELOPERS
---------------------------------------------------------
- Parse the "blocks" array sequentially.
- For each block, switch on "type" and render the corresponding UI component.
- For inline arrays, iterate and render each node according to its type.
- Maintain a global glossary map and make "term" nodes interactive.
- Task blocks with "level" > 1 must be nested inside the parent's "content" 
  to achieve correct indentation – the renderer relies on DOM nesting, not 
  explicit margins.
- key_ideas_box concepts may use either "matrix" (simple) or "content" 
  (flexible). If "content" is present, render its blocks directly; otherwise 
  render "matrix" rows as a table.
- property_matrix rows can contain inline text, a LaTeX string, or language-
  specific LaTeX translations.
- Adapt layout (margins, columns) based on screen size but keep the logical 
  order intact.
